# 省份数量统计工具

基于两种经典算法（深度优先遍历 DFS、并查集 Union-Find）实现无向图连通分量计数，专用于解决「省份数量统计」问题的高性能、工程化实现。

## 问题描述
给定 `n` 个城市和一个 `n×n` 的邻接矩阵 `isConnected`，其中：
- `isConnected[i][j] = 1`：表示城市 `i` 和城市 `j` 直接相连
- `isConnected[i][j] = 0`：表示城市 `i` 和城市 `j` 不直接相连

「省份」的定义是由直接/间接相连的城市组成的集合，本工具的核心目标是计算省份的总数（等价于统计无向图中的连通分量个数）。

## 实现细节
本项目提供两种高效解法，均遵循清晰的工程实践规范：

### 1. 深度优先遍历（DFS）
- **核心逻辑**：递归遍历每个连通分量的所有节点，通过访问标记数组避免重复统计；未被访问的节点即为新省份的起点，每发现一个新起点则省份数加 1。
- **关键优化**：
  - 引入 `visited` 数组标记访问状态，避免同一省份被重复计数；
  - 递归遍历覆盖所有间接相连的城市，确保单个连通分量被完整标记。

### 2. 并查集（Union-Find/DSU）
- **核心逻辑**：通过「查找（Find）」和「合并（Union）」操作管理城市的连通关系，最终统计根节点（父节点等于自身的节点）数量，即为省份总数。
- **关键优化**：
  - 查找操作中加入路径压缩，将查找时间复杂度降至近似 O(1)；
  - 利用邻接矩阵的对称性，仅遍历 `j > i` 的区域，避免重复合并（如 (i,j) 和 (j,i) 仅处理一次）。

## 复杂度分析
| 算法       | 时间复杂度            | 空间复杂度 |
|------------|-----------------------|------------|
| DFS        | O(n²)                 | O(n)       |
| 并查集     | O(n² × α(n))          | O(n)       |

- `α(n)`：阿克曼函数的反函数，增长速度极慢（工程实践中可视为常数）；
- `n`：城市数量（即邻接矩阵的维度）。

## 构建与运行
### 环境要求
- C++11 及以上版本编译器（GCC/Clang/MSVC 均可）；
- CMake 3.10 及以上版本。

### 构建步骤
```bash
# 创建编译目录（规范的工程化编译方式）
mkdir build && cd build

# 生成编译配置文件
cmake ..

# 编译项目
make
